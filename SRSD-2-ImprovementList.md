
# 改进项列表

## 通用

1. 所有页面的新建按钮统一放到左侧，和筛选栏在一行
2. 所有组件的宽度调整一下，宽度要合适，尽量不要换行

## Token 管理

3. Token 管理页面的背景提示分成了两栏，请合并为一栏，格式调整美观一些
4. Token 列表缺少状态字段

## 服务列表

5. 服务列表，鉴权这一列展示：未开启、Basic、Forward
6. 代理地址列展示 http/https，点击可以打开小弹窗，展示默认代理地址和别名代理地址（加上 http 或者 https，支持复制）

## 服务详情

7. 服务详情页的很多 Tab，缺少展示内容，需要补充完整

### 安全配置

8. 服务安全配置 - 服务保护规则，补充 ua-restriction 插件和 uri-blocker 插件
9. 服务安全配置 - 鉴权，选择 Forward 配置时：
   - 配置转发请求头和响应回写头时，用户输入字段名和字段解释，可以添加多组，无默认值
   - 去掉鉴权超时配置

### 健康管理

10. 健康管理配置字段较多，补充完整

### 访问方式

11. 查看 Forward 鉴权服务的访问方式时，展示已配置的转发请求头和响应头回写的字段、解释内容

### 服务别名

12. 服务别名，给用户提示：添加服务别名后，需要在自己的 DNS 里配置 CNAME 指向此服务的代理地址

## 虚拟服务

13. 虚拟服务的流量规则，参考 [APISIX traffic-split 文档](https://apisix.apache.org/zh/docs/apisix/plugins/traffic-split/)；据后端研发反馈，默认后端只能选一个服务，权重不在默认后端这里配置
14. 虚拟服务与普通服务一样有详情页，Tab 较多，区别是没有：实例列表、流量调度、健康管理，需确认并补充其余 Tab 的展示内容

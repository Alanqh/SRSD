# 审核列表 交互规范

> 对应页面：`review-list.html` · 页面类型：列表管理页

---

## 1. 页面概览
- 页面入口：`review-list.html`
- 页面类型：列表管理页
- 核心功能：展示待当前用户（服务持有人/管理员）审核的申请，支持通过/拒绝操作；支持按状态/关键词筛选

## 2. 布局结构

## 4. 数据模型

与申请列表数据模型相同（详见 `apply-list-fe-spec.md`），存储 key：`srsd_reviews`

## 5. 交互流程

### 5.1 搜索筛选
1. 用户选择状态或输入关键词（目标服务/申请人/申请理由）
2. 点击「查询」或按回车 → 表格过滤，分页重置为第 1 页
3. 点击「重置」→ 条件清空，表格恢复全量数据

### 5.2 审核申请
1. 点击状态为"审核中"的记录对应的「审核」→ 审核弹窗打开
2. 弹窗上半部分展示申请摘要（申请ID/目标服务/客户端/申请人/申请理由/申请时间，只读）
3. 下半部分为"审核备注"文本框（选填）
4. 点击「通过」→ 该申请状态变为"已通过"，reviewedAt 更新为当前时间 → 弹窗关闭 → Toast "已通过该申请"
5. 点击「拒绝」→ 该申请状态变为"未通过"，reviewedAt 更新为当前时间 → 弹窗关闭 → Toast "已拒绝该申请"
6. 点击「取消」或弹窗遮罩 → 弹窗关闭，申请状态不变

### 5.3 查看详情
1. 点击某行「查看」→ 查看弹窗打开，展示所有字段（只读）
2. 点击「关闭」或弹窗遮罩 → 弹窗关闭

## 6. 前端校验规则

无（审核操作无必填字段，备注为选填）

## 7. 验收标准

- [ ] 操作：选择状态"审核中"查询 → 预期结果：只显示审核中的记录
- [ ] 操作：已通过/未通过的记录 → 预期结果：操作列不显示「审核」按钮
- [ ] 操作：点击「审核」→ 预期结果：审核弹窗打开，申请摘要正确展示
- [ ] 操作：在审核弹窗点击「通过」 → 预期结果：该记录状态变为"已通过"，审核时间更新，Toast "已通过该申请"
- [ ] 操作：在审核弹窗点击「拒绝」 → 预期结果：该记录状态变为"未通过"，审核时间更新，Toast "已拒绝该申请"
- [ ] 操作：点击「取消」关闭审核弹窗 → 预期结果：申请状态不变
- [ ] 操作：点击「查看」→ 预期结果：查看弹窗展示该申请的所有字段
- [ ] 操作：审核时间为 null 的记录 → 预期结果：表格中审核时间列显示"--"

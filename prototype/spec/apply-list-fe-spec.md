# 申请列表 交互规范

> 对应页面：`apply-list.html` · 页面类型：列表管理页

---

## 1. 页面概览
- 页面入口：`apply-list.html`
- 页面类型：列表管理页
- 核心功能：展示当前用户提交的服务访问申请，支持新建申请、查看详情；状态为"未通过"时可重新申请

## 2. 布局结构

```
┌────────────────────────────────────────────────────────────┐
│ 面包屑：服务注册与发现 / 申请列表                             │
│ 页面标题：申请列表                           [+ 新建申请]    │
├────────────────────────────────────────────────────────────┤
│ 状态[▼]  关键词[______]  [查询][重置]                        │
├────────────────────────────────────────────────────────────┤
│ 申请ID  目标服务  客户端  类型  状态      申请人  申请时间  审核时间  操作    │
│ ──────────────────────────────────────────────────────     │
│ 41  lgh-service-test2  lgh-new  服务  [已通过]  吕青华  …  …  查看   │
│ 37  lgh-service-test2  lgh...   服务  [未通过]  吕青华  …  …  查看 重新申请 │
│ ...                                                        │
├────────────────────────────────────────────────────────────┤
│ 共 N 条                              ‹ 1  2 ›              │
└────────────────────────────────────────────────────────────┘
```

> 新建申请弹窗、查看详情弹窗为浮层

## 3. 组件清单

| 组件 | 职责 | 备注 |
|------|------|------|
| 搜索栏 | 按状态/关键词筛选 | 回车触发查询 |
| 数据表格 | 展示申请记录 | 含彩色状态标签 |
| 操作列 | 查看 / 重新申请（未通过时） | |

## 4. 数据模型

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 唯一标识，由 `genId()` 生成 |
| seqId | number | 是 | 业务申请序号（递增整数） |
| target | string | 是 | 目标服务名称 |
| client | string | 是 | 客户端名称 |
| clientType | `'service' \| 'user'` | 是 | 客户端类型：服务 / 用户 |
| status | `'approved' \| 'rejected' \| 'pending'` | 是 | 已通过 / 未通过 / 审核中 |
| applicant | string | 是 | 申请人，只读（当前登录用户） |
| reason | string | 是 | 申请理由 |
| appliedAt | string | 是 | 申请时间，ISO 8601，只读 |
| reviewedAt | string \| null | 否 | 审核完成时间，审核中时为 null |

## 5. 交互流程

### 5.1 搜索筛选
1. 用户选择状态或输入关键词（目标服务/申请人/客户端）
2. 点击「查询」或按回车 → 表格按条件过滤 → 分页重置为第 1 页
3. 点击「重置」→ 条件清空，表格恢复全量数据，分页重置

### 5.2 新建申请
1. 点击「+ 新建申请」→ 弹窗打开，标题"新建申请"，所有字段清空
2. 用户选择目标服务（必填）、填写客户端名称（必填）、选择客户端类型（必填）、填写申请理由（必填）
3. 点击「提交」→ 触发字段校验
4. 校验失败 → 对应字段显示错误提示，弹窗不关闭
5. 校验通过 → 新申请（状态"审核中"）出现在列表第一行 → 弹窗关闭 → Toast "申请已提交，等待审核"

### 5.3 重新申请
1. 点击状态为"未通过"的记录对应的「重新申请」→ 弹窗打开，预填该记录的目标服务/客户端/客户端类型
2. 用户修改或重新填写申请理由，点击「提交」→ 流程同新建申请

### 5.4 查看详情
1. 点击某行「查看」→ 查看弹窗打开，展示该申请的所有字段（只读）
2. 点击「关闭」或弹窗遮罩 → 弹窗关闭

## 6. 前端校验规则

| 字段 | 规则 | 错误提示 |
|------|------|----------|
| target | 必填，从下拉选择 | "请选择目标服务" |
| client | 必填，trim 后不为空 | "客户端不能为空" |
| clientType | 必填，从下拉选择 | "请选择客户端类型" |
| reason | 必填，trim 后不为空 | "申请理由不能为空" |

## 7. 验收标准

- [ ] 操作：选择状态"未通过"查询 → 预期结果：只显示未通过的记录
- [ ] 操作：输入关键词查询 → 预期结果：只显示目标服务/申请人/客户端包含关键词的记录
- [ ] 操作：新建申请时目标服务未选择点「提交」 → 预期结果：显示"请选择目标服务"，弹窗不关闭
- [ ] 操作：成功提交新申请 → 预期结果：新记录出现在列表第一行，状态为"审核中"，Toast "申请已提交，等待审核"
- [ ] 操作：点击未通过记录的「重新申请」→ 预期结果：弹窗打开，目标服务/客户端/客户端类型已预填
- [ ] 操作：已通过/审核中的记录 → 预期结果：操作列不显示「重新申请」按钮
- [ ] 操作：点击「查看」→ 预期结果：详情弹窗展示该记录所有字段，包括状态和申请理由
- [ ] 操作：审核时间为 null 的记录在查看弹窗中 → 预期结果：审核时间显示"--"

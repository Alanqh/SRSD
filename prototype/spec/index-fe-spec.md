# 概览（首页）交互规范

> 对应页面：`index.html` · 页面类型：大盘页

---

## 1. 页面概览
- 页面入口：`index.html`（模块默认落地页）
- 页面类型：大盘页
- 核心功能：展示服务注册与发现模块的全局统计指标、快捷导航入口，以及最近申请记录

## 2. 布局结构

```
┌────────────────────────────────────────────────────────────┐
│ 面包屑：概览                                                │
│ 页面标题：服务注册与发现                                     │
├─────────────┬──────────────┬──────────────┬────────────────┤
│  命名空间数  │  服务总数    │  我的申请    │  待审核申请     │
│     9       │     10       │     4        │     3          │
├─────────────┴──────────────┴──────────────┴────────────────┤
│ 快捷入口                                                    │
│  [🗂️ 命名空间]  [🔧 服务管理]  [📋 申请管理]  [✅ 审核管理] │
├────────────────────────────────────────────────────────────┤
│ 最近申请                                      [查看全部]    │
│ 申请ID  目标服务  客户端  状态      申请时间                 │
│ 41      lgh-service-test2  lgh-new  [已通过]  …            │
│ ...（最近 5 条）                                            │
└────────────────────────────────────────────────────────────┘
```

## 3. 组件清单

| 组件 | 职责 | 备注 |
|------|------|------|
| 统计卡片行 | 展示 4 项关键指标，可点击跳转对应列表页 | 数值从 sessionStorage 实时读取 |
| 快捷入口卡片组 | 4 个功能模块的快捷跳转入口 | 悬停有阴影效果 |
| 最近申请表格 | 展示最近 5 条申请记录 | 只读，含状态标签 |

## 4. 数据模型

| 指标 | 数据来源 | 说明 |
|------|----------|------|
| 命名空间数 | `srsd_namespaces` sessionStorage | 列表总条数 |
| 服务总数 | `srsd_services` sessionStorage | 列表总条数 |
| 我的申请 | `srsd_applies` sessionStorage | 列表总条数 |
| 待审核申请 | `srsd_reviews` sessionStorage | status 为 `pending` 的条数 |

最近申请数据取 `srsd_applies` 的前 5 条（按 sessionStorage 存储顺序，即最新的在前）

## 5. 交互流程

### 5.1 页面初始化
1. 从各 sessionStorage 中读取数据，计算统计指标
2. 渲染统计卡片、快捷入口、最近申请表格

### 5.2 统计卡片跳转
1. 点击任意统计卡片 → 跳转到对应列表页
   - 命名空间数 → `namespace-list.html`
   - 服务总数 → `service-list.html`
   - 我的申请 → `apply-list.html`
   - 待审核申请 → `review-list.html`

### 5.3 快捷入口跳转
1. 点击任意快捷入口卡片 → 跳转到对应功能页面

### 5.4 最近申请
1. 展示最近 5 条申请（只读），含状态标签
2. 点击「查看全部」→ 跳转到 `apply-list.html`

## 6. 前端校验规则

无（大盘页为只读展示）

# 服务列表 交互规范

> 对应页面：`service-list.html` · 页面类型：列表管理页

---

## 1. 页面概览
- 页面入口：`service-list.html`
- 页面类型：列表管理页
- 核心功能：展示所有服务，支持按命名空间/业务线/关键词筛选，支持新建、编辑、删除服务，点击服务名跳转详情页

## 2. 布局结构

```
┌────────────────────────────────────────────────────────────────┐
│ 面包屑：服务注册与发现 / 服务列表                                │
│ 页面标题：服务列表                                   [+ 新建]   │
├────────────────────────────────────────────────────────────────┤
│ 命名空间[▼]  业务线[▼]  关键词[______]  [查询][重置]             │
├────────────────────────────────────────────────────────────────┤
│ 服务名  命名空间  业务线  代理地址  鉴权  描述  创建时间  操作    │
│ ─────────────────────────────────────────────────────────      │
│ vip  domeos-pub  北京研发中心  vip-domeos...  --  --  …  详情 编辑 删除 │
│ ...                                                            │
├────────────────────────────────────────────────────────────────┤
│ 共 N 条                                    ‹ 1  2 ›            │
└────────────────────────────────────────────────────────────────┘
```

> 新建/编辑弹窗宽度 580px；删除确认框为浮层

## 3. 组件清单

| 组件 | 职责 | 备注 |
|------|------|------|
| 搜索栏 | 按命名空间/业务线/关键词筛选 | 回车触发查询 |
| 数据表格 | 展示服务列表 | 服务名为可点击链接 |
| 操作列 | 详情/编辑/删除按钮 | |
| 分页组件 | 翻页 | 每页 10 条 |
| 新建/编辑弹窗 | 表单提交 | 包含开关组件（鉴权/日志收集） |
| 删除确认框 | 二次确认 | |
| Toast | 操作反馈 | |

## 4. 数据模型

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 唯一标识，由 `genId()` 生成，只读 |
| name | string | 是 | 服务名，创建后不可修改；格式：小写字母开头，含小写字母/数字/连字符，不以连字符结尾 |
| ns | string | 是 | 所属命名空间 |
| bizline | string | 否 | 业务线，由命名空间自动映射 |
| proxy | string | 是 | 代理地址，自动生成：`{name}-{ns}.domeos.sohucs.com` |
| auth | `'none' \| 'disabled' \| 'enabled'` | 是 | 鉴权状态：`none`=无配置，`disabled`=已关闭，`enabled`=已开启 |
| addrType | `'ip' \| 'domain' \| 'any'` | 是 | 实例地址类型 |
| protocol | `'http' \| 'https' \| 'grpc'` | 是 | 协议类型 |
| logEnabled | boolean | 是 | 是否开启日志收集 |
| token | string | 否 | 网关 token |
| desc | string | 否 | 服务描述 |
| createdAt | string | 是 | 创建时间，ISO 8601，只读 |
| updatedAt | string | 是 | 最后更新时间，ISO 8601 |

## 5. 交互流程

### 5.1 搜索筛选
1. 用户选择命名空间或业务线，或输入关键词
2. 点击「查询」或按回车 → 表格过滤 → 分页重置为第 1 页
3. 点击「重置」→ 条件清空，表格恢复全量数据，分页重置为第 1 页

### 5.2 新建服务
1. 点击「+ 新建」→ 弹窗打开，标题"新建服务"，所有字段清空
2. 用户填写各必填字段；鉴权开关默认关闭，日志收集默认开启
3. 点击「确定」→ 触发字段校验
4. 校验失败 → 对应字段显示错误提示，弹窗不关闭
5. 同命名空间下服务名重复 → 显示"该命名空间下服务名已存在"，弹窗不关闭
6. 校验通过 → 新服务出现在列表第一行，代理地址自动生成 → 弹窗关闭 → Toast "创建成功"

### 5.3 编辑服务
1. 点击某行「编辑」→ 弹窗打开，回填数据；服务名不可修改
2. 用户修改命名空间、鉴权、日志收集、token、描述等字段
3. 点击「确定」→ 触发校验 → 通过后列表更新 → 弹窗关闭 → Toast "编辑成功"
4. 点击「取消」或弹窗遮罩 → 弹窗关闭，列表不变

### 5.4 跳转详情
1. 点击表格中的服务名链接 → 跳转到 `service-detail.html?id={id}`
2. 点击操作列「详情」按钮 → 跳转到 `service-detail.html?id={id}`

### 5.5 删除服务
1. 点击某行「删除」→ 弹出确认框："确认删除服务「[name]」？删除后无法恢复。"
2. 点「确定」→ 该服务从列表移除 → Toast "删除成功"
3. 点「取消」→ 列表不变

## 6. 前端校验规则

| 字段 | 规则 | 错误提示 |
|------|------|----------|
| ns | 必填，从下拉选择 | "请选择命名空间" |
| name | 必填；以小写字母开头，含小写字母/数字/连字符，不以连字符结尾；同命名空间内不重复 | "服务名不能为空" / "以小写字母开头，可包含小写字母、数字和连字符，不能以连字符结尾" / "该命名空间下服务名已存在" |
| addrType | 必填，从下拉选择 | "请选择实例地址类型" |
| protocol | 必填，从下拉选择 | "请选择协议类型" |
